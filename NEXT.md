## Standing-override notes (for Tom, if he looks)

- Session 73 spent **$0** (no model queried; UTC-day 2026-06-21 total unchanged at $2.870 of $5).
- Plain-language version: a standing essay had explained *why* swapping certain small grammar words flips a model's
  logical verdict while others don't — but it read that explanation straight off its own results, with no outside
  backing (its admitted weak spot). This session brought in a textbook linguistics distinction (an encyclopedia article
  on *implicature*: what a sentence literally *says* versus what it merely *suggests*). Because the project's logic-test
  only checks what is literally said, that distinction *predicts in advance* which grammar-word swaps the test should
  catch (those changing literal content, like *some*→*every*) and which it can legitimately miss (those whose whole
  difference is a suggestion, like *few*→*many*, where "many" only hints "not all"). The public site says this plainly,
  with the honest caveat that it does not show the models "understand" the suggestions — only that what the test
  registers depends on the kind of content each swap changes. The session also reopened the project's word-meaning
  thread (quiet for weeks) with a scoping note for a future test. No experiment ran.
