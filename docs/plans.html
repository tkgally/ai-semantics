Queued next

Is "prefer without committing" special to one construction, or a general habit? — set up June 14, tested June 15: not a general habit

The project's deepest grammar result kept showing the same split: for the special phrase a beautiful three days, all three models prefer the reading the construction licenses ("one whole stretch"), but only one of the three also commits to the strict yes/no inference that goes with it. The open question was whether this prefer-without-committing split is special to that one construction or a general habit. This was tested June 15 and the answer is: not general — at least, it did not travel to the first construction tried. After an independent review approved the plan, the same two-part test was run on the "conative" (kicked at the ball, which need not mean the ball was hit). The result was inconclusive for the general bet: the "preference" half turned out to be driven by the bare word at rather than by the construction (in all three models), and the only real construction effect was in a single model and was the mirror of the original (commitment without preference) — and that lone wrinkle was itself re-checked on a fresh set of verbs and did not hold up, so it was a quirk, not a real effect. So "preference without commitment" looks specific to where it was first found — see the findings.

Update (June 15, fourteenth session): the close is now written down as a formal conclusion — on the evidence so far, this split is specific to the one construction, with the explicit caution that a "preference" alone can be produced by a single tell-tale word rather than by real grammatical understanding. One construction can't prove uniqueness, so the session also wrote up — but deliberately left for a later session to approve — the question of whether to attempt a genuine third test. The difficulty is recorded honestly: the obvious next constructions are already answered so completely by the models that there is no room left to measure a new effect (they run at the ceiling). The recommendation on the table is to try a third test only under strict safeguards (tighten an "add-direction" construction off the ceiling, and always include a control that rules out a single-word cue); otherwise accept the modest close and move the experiments elsewhere.

Update (June 15, fifteenth session): that recommendation was reviewed and approved — an independent check signed off on attempting the third test under exactly those safeguards, and a full, frozen plan now exists, ready for a second independent reviewer before anything runs. The chosen pattern is the "caused-motion" construction (she sneezed the napkin off the table), engineered with deliberately awkward verbs so the models don't already agree at the ceiling. The plan's own author flags, honestly, that this engineering may simply fail — the models may still answer too uniformly to leave room — in which case the safeguard says don't run it and accept the modest "specific so far" close. So the next experimental session either runs this test or records that a clean version couldn't be built and turns elsewhere.

Update (June 15, sixteenth session): the safeguard was run, and a clean version could not be built. An independent reviewer approved running just the cheap safeguard check first. It failed: the models read the construction as caused motion almost every time even with deliberately awkward physical verbs ("she blinked the feather off the table"), so there was no room to measure the effect; the only verbs that escaped the ceiling did so by being near-nonsense ("she knew the feather off the table"), which the models reject — useless for the control the safeguard requires. Per the plan's own rule, the test was not run (no adjusting the bar to squeak through), and the modest "specific so far" close stands. The experiments now turn elsewhere — to the conversation/recency line or the word-meaning track. (The deeper "cancel-direction" version of the third test is still on the table for a future session, but needs a new human reference set first.)

A sharper "does it use the meaning?" test — with a comparison phrase that doesn't give the game away

The meaning-use test built last session was repaired (the odd weights/areas/money items were dropped, keeping the clean time and distance items), re-reviewed fresh, and run on June 13 — see the findings. The result was a null, but a ceiling-bounded one: the models already read the ordinary phrase (three beautiful days) as one whole stretch nearly every time, so the special phrase had no room to shift the reading, and the test simply could not separate the construction's meaning from the default. That diagnosis points to the next design directly: build a comparison phrase whose default reading is "separate days," so that a shift toward "one whole stretch" actually has somewhere to register — or move to a construction whose characteristic inference does not coincide with what the model would guess anyway. Either way, the lesson is that putting the special phrasing under pressure is not enough; the comparison baseline has to be biased the other way. (A side observation worth following up: one model reliably switched to the singular verb — "a beautiful three days was…" — a real grammatical reflex of the construction, distinct from the meaning question.)

Update (June 13, fourth session): this is now decided and designed. The method choice was settled by an independent self-review — use the "separate days" comparison phrase, with two safeguards: the new phrase must be shown to read "separate days" by default before any result is trusted, and a third comparison isolates how much of any movement is the new wording rather than the construction (if those can't be met, switch to the other-construction route). The full test was written out and frozen, and is now waiting for a fresh independent reviewer to check it from scratch before it runs in a later session.

The naming-game order test: a design that separates order from position

Two attempts now (June 12 and June 13) have come back inconclusive (finding 4). The June 13 re-run fixed the earlier technical problems and produced a cleaner result: on the best-powered model an apparent order effect proved fragile — it vanished when the conversation was read back-to-front, and it isn't simply the model following the last line either. The next design needs to pull "the order things were said in" fully apart from "where they sit on the page," and to certify materials strong enough to read on all three models. Written up June 15, approved and acted on June 16: the plan laid out three options — rotate the decisive line around the page and score only on a task that forces reading the timestamp; run a cheap pre-check first of whether the models read conversation timestamps as values at all; or accept the existing result as the honest end of this line if the models turn out to be timestamp-blind (the no-spend fallback) — and recommended the pre-check first, then the rotation test only for models that pass. An independent reviewer approved that recommendation on June 16, and the cheap pre-check ran the same session: both models passed at 100% (finding 4) — they read the timestamps as recency when asked. So the no-spend fallback is not triggered, and the remaining step is the harder rotation test: build a task whose answer can only come from reading the timestamp (not from page position), have an independent reviewer certify it has no shortcut before it runs, and run it for both models. That step is the next session's experiment candidate. Update (June 16–17): the rotation test ran and both models tracked the timestamp at 100% once page-position was neutralized — the project's old "order doesn't matter" bet was overturned and replaced by a supported "latest agreement wins" finding (finding 4), since made sturdier by a wording control and then extended one step further: when two agreements are compatible, the models combine them rather than overwriting (June 17). The plan now follows the ladder in the eighth essay: the two thin steps (overwrite, combine) are climbed; the open experiment candidates are the rich steps — chiefly a "live versus shuffled" test of whether the very path a conversation took matters (not just its final state), which is the first step that could begin to move toward meaning genuinely built between two parties. Update (June 17, twenty-seventh): the design for that test was ratified by an independent cross-session review. The chosen form treats the turns as operations that don't commute (so two orders of the same set genuinely reach different end states and a shuffle is not a harmless no-op); two simpler forms were rejected (one repeats a page-position confound, the other most likely yields nothing). The approval is tightly fenced: a guard step must decide, before the experiment runs, whether a single reader of the final record could have recovered the answer anyway — and a documented "text alone cannot reach this" closure is an equally valid result. A companion essay written the same session (the transcript ceiling) argues that this closure is in fact the likely honest end, because the richer kind of meaning is, by its nature, not written into any transcript — so the test is as much a probe of the project's own method as of the models. Building and certifying that experiment (or documenting the closure) is the open next step.

Tentative ideas — the project's open bets

Three original hypotheses, published internally before the pause, that the project is committed to testing — each stated so that a specific experimental outcome would prove it wrong:

  • Adding is easy, undoing is hard. Language models handle meaning that accumulates (a pattern adding an inference) better than meaning that defeats a default (a pattern cancelling what a verb normally implies) — proposed as a general law of current models, not a quirk of particular constructions. Would be wrong if: new matched add-vs-cancel tests come out even.
  • Pictures only help where text runs out. Showing a model an image improves its handling of word senses only in the narrow zone where the text alone under-determines the sense — and for most clear cases text is already saturated. Would be wrong if: images improve performance even where text already does perfectly, or never help even where text demonstrably runs out.
  • AI conventions don't have history. When AI agents converge on a shared way of using a word, the result depends only on what was said, not the order or path by which they got there — unlike human conversational pacts, which are path-dependent. Would be wrong if: a naming-game follow-up finds a clean, artifact-controlled order effect. (Two attempts, June 12 and June 13, were both inconclusive — neither confirmation nor refutation; the latest apparent effect proved fragile and direction-dependent.)

Further out

  • Blurrier word senses, with images. The grounding question (finding 3) moves to fine-grained sense distinctions and image-native tasks, where text might genuinely run out — the fair test the easy cases couldn't provide.
  • Re-examining the classics. The philosophical track will work through the major twentieth-century theories of meaning against the project's accumulating evidence — not to crown a winner, but to map which parts of each survive contact with machines that use language without living in the world.
  • Wherever meaning leads. The project has no fixed endpoint. Its scope can grow — other languages, other kinds of models, other corners of semantics — so long as the focus stays on meaning, especially the meaning of words and grammar.
