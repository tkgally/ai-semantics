1. **Overall: NO-GO.** The build is close, but I would not freeze-run it without at least one additional safeguard because the swap logic is **not demonstrably contrast-preserving in all retained paradigms**, and one of the core representative cases already shows a clear polarity-risk pattern. 2. **Faithful preservation of the minimal contrast?** **Not fully.** Main concerns: - **determinizer–noun agreement paradigms:** swapping the noun while preserving number can still break the intended contrast if the substituted noun changes **countability, article compatibility, or lexical plurality behavior** in ways that interact with the agreement manipulation. - **subject–verb agreement with proper-name swaps:** the special rule “not the pre-verb subject” is good, but the representative example shows that swapping the object name can create **selectional oddity**; more importantly, if any retained item has a name-like subject elsewhere, title-case is not enough to guarantee number/interpretation stability. - **sentential negation / NPI scope and only-NPI scope:** these are the most fragile. Even if the swapped token is outside the locus, changing an embedded noun/name/adjective can alter **scope preferences, discourse prominence, or locality judgments** enough to affect acceptability asymmetrically. This is a real polarity-flip risk not guaranteed to be caught by the mechanical checks. - **superlative-quantifier items:** adjective/noun swaps inside quantifier environments can change **semantic comparability, classifier-like interpretation, or NP internal well-formedness**. Again, grammaticality can shift without changing the token-difference count. So: the revalidation is necessary, but it does **not** fully certify grammatical-minimality preservation. 3. **“Semantic oddity cancels in the 2AFC” correct?** **Only partly.** It is true that if the same swapped content appears in both members of a pair, some item-level semantic weirdness can cancel **in expectation**. But it can still bias Delta-acc if the oddity: - interacts differently with the **good** vs **bad** member, - changes **parsing difficulty** asymmetrically, - triggers **agreement/scope repair** tendencies in one member more than the other, - or causes the model to rely on **lexical priors** that differ across the two constructions. So the claim is **not safe as a blanket guarantee**. 4. **Residual anti-cheat concern?** **Yes.** The most obvious tunable concern is the **selection of “swappable” items by swappability score** and then using the most permissive, high-coverage subset. That can indirectly optimize for a desired outcome by preferentially keeping items that are easiest to preserve or most likely to produce a particular effect. Also, the **<= / >= thresholding around coverage and dropped paradigms** could be used strategically to steer which paradigms survive into the final verdict. 5. **One freeze-time condition I would add:** Require a **manual audit of a random sample from each retained paradigm, especially all deep/scope paradigms, with explicit certification that the swapped item preserves POS, number, and NP-internal well-formedness and does not alter scope/attachment plausibility**. If you want a stricter single rule: **drop any paradigm where swapped items include semantically anomalous or selectionally incompatible nouns/names in >5% of pairs.**
