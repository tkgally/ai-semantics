**Overall:** **GO-WITH-CONDITIONS** ### 1) Per-gate vote - **Q1: B** — Best tradeoff: enough strata to test the depth hypothesis, but frozen by a prewritten locality rule so it doesn’t become post hoc cherry-picking. I would only accept it if the rule and included paradigms are locked before model calls. - **Q2: A** — Necessary. Single-order forced choice is too exposed to position bias and order effects; dual-order + netting is the right behavioral control. - **Q3: A** — Acceptable if framed as **relative** evidence only. Using per-paradigm human agreement as the anchor is the right yardstick for “hard for models where hard for humans”; contamination makes absolute accuracy only an upper bound. ### 2) Single biggest methodological risk **Curated subset bias / researcher degrees of freedom in Q1.** A 9–12 paradigm depth-stratified subset can silently become a “best story” set unless the locality rule is genuinely ex ante, mechanistic, and non-outcome-contingent. If the subset is chosen after seeing pilot behavior, the human-anchored profile alignment is not trustworthy. ### 3) Freeze-time condition I would add **Lock the subset-selection rule in writing before any model outputs, and require a public mapping from each candidate paradigm to stratum based only on syntactic/structural criteria.** Concretely: define the rule so that inclusion is determined without reference to expected model performance or human-agreement values, then publish the final selected paradigms and the excluded paradigms with reasons. ### 4) Any place the design looks aimed at a desired result? **Potentially yes: Q1.** The default to a depth-stratified selected subset could be viewed as steering toward a “shadow-depth” narrative unless the inclusion rule is strictly ex ante and fully agnostic to outcomes. Q2 and Q3 look more like defensible controls than result-seeking.
