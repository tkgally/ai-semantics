VERDICT: **NO-GO**

- **BLOCKER — Outcome space is not genuinely symmetric; NULL is partly pre-baked.**  
  You define NULL as “quant cell no longer LOWEST of four classes for ≥2/3 models,” which depends on the same class structure that motivated the probe and privileges one specific failure mode. **Fix:** pre-register NULL as a broader family of non-inversion outcomes (e.g., not uniquely class-low, or not below baseline by a margin), and keep it logically independent of the class-ranking narrative.

- **BLOCKER — Q2 thresholding is arbitrary and unstable.**  
  The 0.70/0.30 cutoffs plus bottom-3 drop rule are too bespoke and can manufacture CLASS/LEXICAL/MIXED splits from the same data. **Fix:** define the primary decision rule as a single monotone criterion on the distribution of per-modifier contrasts, then make the 0.70/0.30 scheme secondary/descriptive.

- **BLOCKER — B_m risks anchoring the test to the comparison set rather than the construct.**  
  “Mahowald-own non-quant reference on the same occasion” is better than a stale baseline, but if the reference set is selected/filtered to be temporally interpretable, it can still encode the desired ordering. **Fix:** pre-specify the exact reference items, include both positive and negative temporal controls, and require that B_m be computed from the full fixed set with no post hoc trimming.

- **SHOULD-FIX — Arm 1 / Arm 2 separation is mostly sound, but the hybrid reporting needs a stricter firewall.**  
  Item-matched human comparison for anchored items is legitimate; internal-contrast for novel items is also fine. But the C-HYBRID default can invite cross-arm interpretive blending. **Fix:** state that Arm 1 can only validate anchored items and Arm 2 can only support internal-class claims; no pooled human-vs-model inference across arms.

- **SHOULD-FIX — “All outcomes reachable” is not quite true under the current subtype clause.**  
  The “or if the inverters are predominantly one polarity subtype” language can swallow cases that would otherwise look CLASS or MIXED, making the taxonomy order-dependent. **Fix:** define subtype as a post hoc descriptive annotation, not a vetoing clause, or formalize it as an explicit tie-break only after the main decision rule fires.
