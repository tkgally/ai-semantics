BUILD-v2 (session 69). Curated carriers, signed-+1 length-matched content swaps. {fw}/{Fw}=function word (premise only; Fw is sentence-initial capitalized form for quantifier arms with cap:true); {cw}=content word (premise AND hyp). base/fn = predicted NLI label (0 ent, 1 neu, 2 con). pairs=[out,in]; build.py drops any pair failing freq(b,c)+SIGNED-length(d) matching against its function pair. ONE in-word per (frame,out-word) to avoid double-counting the function arm. Authored + grammar/coherence/flip-checked by the lead agent. Function arms: because (subordinator causal->concessive), some (quantifier existential->universal, singular), will (modal future->conditional), few (quantifier paucal->multal, plural -- the build-v2 inventory-widening pair; many->every was rejected for breaking number agreement). Content semantic classes: noun_person, noun_thing, verb, adj. Content-pair frequencies are SUBTLEX-US Lg10WF; each content swap matched to its FUNCTION pair within |dLg10WF|<=0.10 at both ends and signed dlen == function dlen (+1). NOTE: faithful matching keeps per-pair content diversity uneven (the because/some/will arms rest on few out-words each; the few->many arm is the diverse, high-supply arm) -- see matching-report.json (condition 6).
