# predictions.md — the registered-bet ledger

**What this is.** One scoreboard for the project's registered predictions — conjecture
confirmation/falsification criteria, essay revision triggers, theory-page predicts/forbids —
with each one's current status. The project already registers bets and honors outcomes; this
page makes the calibration record *visible in one place* (adopted 2026-07-02, program item B5,
[`decisions/resolved/program-2026-07-adoption`](decisions/resolved/program-2026-07-adoption.md)).

**Status vocabulary.** `open` (registered, not yet tested) · `fired-for` (evidence landed on
the bet's side) · `fired-against` (the bet lost; the page records the loss) · `discharged`
(the trigger/criterion was consumed — a conjecture tested, a trigger fired and the revision
made) · `retired` (withdrawn for stated reasons other than a test).

**Maintenance (PROTOCOL §3).** Registering a bet adds a row *in the same session*; any firing,
discharge, or retirement updates the row in the session that lands the evidence. Every row
must be verifiable against its source page — **no row is ever added from memory, and none is
retro-fitted**. Rows are never deleted; outcomes accumulate. The registering page stays
authoritative; the row compresses.

## What earns a scored row — the scope rule

A registered trigger/criterion is **scored here** when it is a *bet*: it predicts a side on
the outcome of a test the project can itself run or has run (a probe, replication, control,
build, or a reading of an in-repo source), **or** it has already resolved on landed evidence.
Firing depends on evidence the project gathers about model behaviour, its own instruments, or
its own source-characterisations.

A trigger is **not scored here** — it is a genuine revision trigger but a *standing contingency*
listed in §C — when its firing awaits an **external event the project does not control**
(ingestion of a not-yet-in-repo primary source, release of an external human resource,
availability of a representational/mechanistic method the panel lacks, a Tom rule-amendment) or
when it is a pure **self-discipline** guard ("if a later page over-claims this as proven law").
These live in full on their essay/theory pages, which stay authoritative — §C names them per
page so the audit trail is complete without turning the scoreboard into a wishlist.

several pages) are one row, with the co-registrations noted. Absence of a row now means *not a
scored bet* (see §C), not *not collected*.

## How to add a row
