# Decision: certifying "matched ambiguity KIND" across levels, and the anchor for a disjunctive lexical class

## Why this exists

The shared-instrument cross-level probe ran and returned **dissolution**
([`result/cross-level-shared-instrument-v1`](../../findings/results/cross-level-shared-instrument-v1.md)):
under one frozen instrument the *discrete-on-the-moment* posture held **only at the lexical
level**, while the same models went *graded-on-the-moment* (took "UNCLEAR" elevated) at the
constructional and relational levels — 3/3 models dissolve. But that result disclosed, before
the run, a **load-bearing comparability residual** that it could not neutralize, and it states
the consequence plainly. Verbatim from its §"The load-bearing caveat — what the dissolution does
and does not show":

> "the stimulus **body** — and with it the **kind of "in-between"** — unavoidably differs:
> - a lexical **bridging** pair is a human-rated **usage-similarity midpoint** (DURel ≈ 2–3): a
>   *graded* similarity the model can still lean on and resolve to same/different;
> - a constructional **ambiguous** item is a *genuine structural* two-reading (both readings fully
>   available);
> - a relational **ambiguous-midrecord** item is a *genuine underdetermination* (a same-round dual
>   binding with no tie-break — both/neither)."

And the crux this gate exists to address, verbatim:

> "Whether the lexical discreteness reflects a deep model fact or merely that a usage-similarity
> midpoint is a *softer* kind of ambiguity than a structural one is **not separable in this
> design** (C3 guards that the instrument reads cleanly on each level's clear controls, but it
> cannot neutralize the body difference …)."

So the dissolution is real **under the frozen instrument**, but the most natural mechanism (the
lexical leg is special because a usage-similarity midpoint is a *softer* ambiguity than a
structural both/neither) is confounded with "the lexical level is special as a deep model fact."
The result already reads this in the **deflationary** direction — and is right to: "Either way the
unified 'one cross-level property' conjecture loses." This gate is the *follow-up that would
separate the two*: it would **equalize the KIND of ambiguity across levels** — construct a lexical
class that is genuinely **disjunctively** ambiguous in a single context (a true two-sense item,
e.g. a balanced homonym context with no contextual tie-break), matched in *kind* to the
structural/relational genuine ambiguities — and re-run the **frozen** shared instrument, to test
whether the lexical commit-without-hedging **survives** a harder (disjunctive) lexical ambiguity
or was an artifact of the milder (scalar-similarity) bridging stimulus.

This is exactly the "future `wiki/decisions/open/` candidate" the open question reserved
([`open-question/gradience-population-not-moment`](../../findings/open-questions/gradience-population-not-moment.md),
§"What would answer this": "the operationalization of such a probe is **not** decided here and
would be a future `wiki/decisions/open/` candidate, with the same freeze-before-results
discipline …"). It inherits the shape and the anti-cheat discipline of the resolved
shared-instrument gate
([`decisions/resolved/cross-level-shared-instrument-operationalization`](cross-level-shared-instrument-operationalization.md))
and the **claim-cap** precedent of the resolved lexical anchor gate
([`decisions/resolved/lexical-bridging-context-anchor`](lexical-bridging-context-anchor.md)).

## What is being decided (scope), and what is not

**In scope (this page):**

- **The matched-kind certification.** How do you certify, *before any model output*, that each
  level's "ambiguous" item is the **same KIND of indeterminacy** — specifically a genuinely
  **disjunctive** both/neither with **no nearer pole to lean to** — so the cross-level
  discrete-vs-graded contrast is *fair* and the dissolution/survival is not an artifact of the
  lexical leg presenting a *softer* (scalar-similarity) ambiguity than the other two?
- **The human-anchor posture for the NEW genuinely-disjunctive lexical class.** A true balanced
  homonym context is **NOT** a DWUG usage-similarity midpoint, so the lexical leg's inherited cap
  (usage-similarity intermediacy) **does not transfer** to it. This new class needs its own
  anchor decision or an `internal-contrast-only` declaration — surfaced here as an explicit
  sub-question, **not** invented.

**Out of scope (other gates / future calls):**

- **The shared commitment instrument itself.** Already resolved (Option A: graded
  SAME/DIFFERENT/UNCLEAR + 0–100 confidence, C1–C4 binding) by
  [`decisions/resolved/cross-level-shared-instrument-operationalization`](cross-level-shared-instrument-operationalization.md).
  This gate changes the **lexical stimulus class**, not the instrument; the instrument and its
  C1–C4 conditions are **inherited unchanged** and re-frozen.
- **Whether to run at all.** A future independent **pre-run critic** GO/NO-GO and a **budget**
  call (charter §8) decide that — not this page. A NO-GO defers the run; it never relaxes a band.

## The first-class NULL / symmetric reporting, declared up front

Per [`CLAUDE.md`](../../../CLAUDE.md) rule 6, the negative is **first-class** and declared
**before** any data. Two outcomes must be reported **symmetrically**, with no after-the-fact
relabelling:

- **Survival** — the lexical commit-without-hedging (near-zero %UNCLEAR on the ambiguous item)
  **persists** even when the lexical item is genuinely disjunctive (balanced homonym, no
  tie-break). This would *strengthen* the "lexical is special as a model fact" reading (the
  softness explanation is ruled out) and would **revive** part of the unified conjecture the
  dissolution defeated — so it must clear an *even higher* anti-cheat bar, not a lower one.
- **Collapse** — the lexical %UNCLEAR **rises** to meet the structural/relational levels once the
  lexical ambiguity is matched in kind. This would *confirm* the deflationary reading the
  dissolution already favored (the lexical "discreteness" was an artifact of the *softer*
  bridging stimulus), and must be reported as cleanly as a survival.

Neither outcome may be reached by re-selecting the disjunctive lexical class against model
confidence (the named anti-cheat violation — see Anti-cheat note). The reading rule below fixes
the survival/collapse threshold **before** the run.

## Q1 — How is "matched ambiguity KIND" certified across levels?

The three options differ in **where** the matched-kind certification lives and **how** the
lexical leg is made disjunctive.

### Option A — Cross-level "no-nearer-pole" certification (one disjunctive item per level, lexical included via balanced-homonym contexts)

- **What.** Certify *each* level's ambiguous item, *before any model call*, as genuinely
  **disjunctive**: two complete readings / bindings are fully available and there is **no graded
  midpoint or nearer pole to lean to**. The constructional and relational legs already satisfy
  this (a structural two-reading; a same-round dual binding with no tie-break, per the result).
  The lexical leg is rebuilt to satisfy it too: a **balanced homonym context** (e.g. a sentence
  where a homonym's two senses are both fully licensed and the surrounding context supplies no
  disambiguating tie-break), replacing the DWUG scalar-similarity bridging item.
- **Certification criterion (frozen):** for every level, an *expert-stipulated* "no-nearer-pole"
  check — both readings independently completable, neither contextually favored — recorded and
  sha256-frozen before the run. The lexical class is certified as a balanced homonym context, not
  a DURel midpoint.
- **Strength.** Directly equalizes the *kind* across all three levels — the cleanest answer to
  the result's own crux. The discrete-vs-graded split, if it persists, can no longer be charged to
  "lexical is softer," because every level is now a genuine both/neither.
- **Weakness (forced).** It **discards** the human-anchored lexical signal the prior leg had: a
  balanced homonym context is **author-stipulated**, so the lexical leg drops from
  human-comparison-capped to (at best) `internal-contrast-only` unless an external homonym-sense
  resource certifies it (see Q2). Expert "no-nearer-pole" stipulation is itself a judgement that
  could be tuned; it must be frozen and, ideally, set by a fixed rule (e.g. drawn from an existing
  homonym inventory), not improvised per item.
- **Gaming/tuning surface.** Choosing *which* homonym contexts count as "balanced" after seeing
  which ones the model hedges on; calling a context "no nearer pole" when it in fact carries a
  weak contextual lean — both forbidden by the freeze.

### Option B — Keep DWUG for the lexical leg, ADD a second balanced-homonym lexical class as a *within-lexical* contrast

- **What.** Do **not** replace the DWUG scalar-similarity bridging leg; **add** a second lexical
  ambiguous class — balanced-homonym contexts (genuinely disjunctive) — alongside it, so the
  design tests **scalar-vs-disjunctive *within* the lexical level**. The within-lexical contrast
  (does %UNCLEAR rise from the scalar-bridging class to the disjunctive-homonym class, for the same
  model under the same instrument?) is the primary test; the cross-level comparison becomes
  secondary.
- **Strength.** *Cleaner than cross-level matching.* It sidesteps the worst comparability problem
  — it never has to certify that a lexical item and a constructional item are "the same kind"
  across radically different stimulus bodies; it only has to certify two **lexical** classes
  differing in kind, holding the level (and most of the stimulus body) fixed. A within-level
  manipulation that stresses the moment pole is exactly the second lever the open question named
  ([`open-question/gradience-population-not-moment`](../../findings/open-questions/gradience-population-not-moment.md),
  §"What would answer this": "A within-level manipulation that stresses the 'moment' pole …"). It
  also **preserves** the DWUG-anchored leg intact (its usage-similarity cap untouched), and isolates
  the *softness* variable with the level held constant — the most direct test of the result's crux.
- **Weakness (forced).** It does **not** fully restore the *cross-level* matched-kind comparison —
  if the disjunctive-homonym lexical class still commits (no %UNCLEAR rise) it is suggestive that
  the lexical level is genuinely special, but the constructional/relational legs are not re-run in
  this design, so the cross-level claim still rests on the prior (unmatched-kind) run for those
  legs. The new disjunctive-homonym class still needs its own anchor posture (Q2).
- **Gaming/tuning surface.** Selecting the homonym set against the model's hedging; choosing the
  scalar-vs-disjunctive boundary after seeing the spread. Both forbidden by the freeze.

### Option C — Defer: declare the kind-confound behaviorally unresolvable at this N, keep the dissolution's deflationary reading without the follow-up

- **What.** Judge that no behavioral design at the available N can cleanly separate "lexical is a
  deep model fact" from "usage-similarity midpoint is a softer ambiguity," and **decline** to run
  the follow-up. The result's dissolution already reads deflationary either way ("Either way the
  unified 'one cross-level property' conjecture loses"), so the project keeps that reading and
  records the kind-confound as a **standing, disclosed residual**, not a runnable question.
- **Strength.** Spends nothing; avoids manufacturing a within-model contrast that — even if it
  "survives" — could not lift the lexical leg above `internal-contrast-only` without an anchor the
  project may not have; honest about the limits of behavioral evidence at small N.
- **Weakness (forced).** Leaves a *named, separable-in-principle* confound unexamined when at least
  one clean within-lexical design (Option B) is available and cheap; "behaviorally unresolvable" is
  a strong claim that Option B's within-level isolation arguably refutes. Risks ossifying a residual
  the project could actually move.
- **Gaming/tuning surface.** None (nothing is run) — but deferral must not be used to *avoid* a
  result that might revive the unified conjecture; the symmetric-reporting rule above is the guard.

## Q2 — The anchor posture for the NEW genuinely-disjunctive lexical class (bound sub-question)

**A balanced homonym context is NOT a DWUG usage-similarity midpoint.** The resolved lexical
anchor gate certified only a *usage-similarity* yardstick and **capped** the claim accordingly —
verbatim from
[`decisions/resolved/lexical-bridging-context-anchor`](lexical-bridging-context-anchor.md):

> "the result may make a **human-comparison claim only about usage-similarity intermediacy** … It
> may **NOT** claim humans certified two senses co-present."

That cap was *designed for the scalar (DWUG) bridging stimulus*. A balanced-homonym disjunctive
context makes a **different** human-grounding demand: it asserts that two senses are genuinely
**co-present and equally licensed** — which is precisely the "two senses co-present" reading the
DWUG route was forbidden from claiming. So the inherited usage-similarity cap **does not transfer**
to the disjunctive lexical class; the new class needs its own anchor decision. The posture options:

- **(Q2-a) Anchor to an existing in-repo / reachable sense-annotated homonym resource**, if one
  bears (e.g. a homonym/sense inventory that certifies two distinct, co-licensed senses for the
  target word). This is the *only* route that could license a human-comparison claim about
  genuine sense co-presence — and it must be a **separate, cross-session-ratified anchor
  decision** of its own, not settled here. **No such resource is asserted to exist by this page**;
  whether one is in-repo or reachable is an explicit open sub-question (see "Anchor / decision
  gap" below). No anchor is invented.
- **(Q2-b) `internal-contrast-only` for the disjunctive lexical class.** Absent a ratified homonym
  sense-anchor, the new lexical class is author-stipulated and makes **no human-comparison claim**
  — its force is a within-model contrast (does *this model* commit or hedge on an item *the author*
  built as balanced-homonym disjunctive). This is the honest fallback and mirrors the
  constructional/relational legs' existing `internal-contrast-only` posture. It is a **strictly
  weaker, different claim** than a human-anchored one and must be labelled as such — and it must
  **never** be used to *dodge* a genuine anchor obligation (the discipline from
  [`CLAUDE.md`](../../../CLAUDE.md) anchor state (b) and the lexical anchor gate's Route 3 note).

Under either Q2 posture, the unified result inherits the **weakest common strength** rule: if any
leg is `internal-contrast-only`, the whole cross-level statement is internal-contrast-only. Option
B preserves the DWUG-capped leg, but its *new* disjunctive class still falls under Q2-b unless
Q2-a is separately ratified, so a within-lexical contrast that mixes a usage-similarity-capped
class and an internal-contrast-only class must be stated at the internal-contrast-only floor for
the disjunctive arm.

## Q3 — The comparability / equalization reading rule (bound, not optional)

The reading rule is fixed **before** any output:

1. **Freeze the matched-kind certification.** The "no-nearer-pole" / balanced-homonym criterion,
   the exact lexical disjunctive item set (ids + the certification record), the per-level
   "ambiguous vs clear" class definitions, and the survival-vs-collapse threshold are
   sha256-frozen before the first probe call. **No per-level instrument-shopping and no
   per-item kind re-certification after any output is seen.**
2. **Inherit the instrument and C1–C4 unchanged.** The shared instrument (Option A: graded
   SAME/DIFFERENT/UNCLEAR + 0–100 confidence) and its binding conditions C1 (categorical decline,
   not confidence, is load-bearing for the moment pole), C2 (numeric/wording sha256 freeze), C3
   (clear-class precondition → per-level NO-GO), C4 (channel-sensitivity guard for confidence) are
   carried over verbatim from
   [`decisions/resolved/cross-level-shared-instrument-operationalization`](cross-level-shared-instrument-operationalization.md)
   and re-frozen. This gate changes **only the lexical stimulus class's KIND**, nothing else.
3. **Pre-commit survival vs collapse.** **Survival** = the lexical disjunctive class's decline
   (%UNCLEAR) stays near-zero and CI-indistinguishable from the prior scalar-bridging lexical leg,
   i.e. it does **not** rise to meet the structural/relational levels. **Collapse** = the lexical
   disjunctive class's decline rises (CI lower bound > 0 over its clear controls) toward the
   structural/relational pattern. Report whichever the frozen design shows, symmetrically.

## Provisional default (NOT a decision)

Recorded for a *later* independent adversarial-review pass; **not decided here**, and **not run**.

- **Q1 — matched-kind certification: PROVISIONAL DEFAULT = Option B** (keep DWUG for the lexical
  leg; **add** a balanced-homonym disjunctive lexical class as a *within-lexical* scalar-vs-
  disjunctive contrast). **Reasons:** (i) it is the **cleanest** isolation of the result's actual
  crux — it varies *only* the ambiguity KIND with the **level held fixed**, so a %UNCLEAR rise (or
  its absence) is attributable to softness-vs-disjunctiveness and not to the unmatched stimulus
  bodies that Option A still has to bridge across levels; (ii) it is exactly the "within-level
  manipulation that stresses the moment pole" the open question already named as a legitimate
  lever; (iii) it **preserves** the human-anchored DWUG leg (its usage-similarity cap intact)
  rather than discarding a real human signal, as Option A would; (iv) it is the **cheapest** clean
  test and avoids the strong, arguably-false "behaviorally unresolvable" claim of Option C. Option
  A is the right choice *if* a full cross-level matched-kind statement is wanted and a homonym
  sense-anchor (Q2-a) is reachable; it is held as the stronger-but-costlier alternative. Option C
  is rejected as the default because Option B refutes its "unresolvable at this N" premise.
- **Q2 — anchor for the disjunctive lexical class: PROVISIONAL DEFAULT = Q2-b
  (`internal-contrast-only`)**, *unless* a future, separately-ratified anchor decision establishes
  a reachable sense-annotated homonym resource (Q2-a). The disjunctive arm makes **no
  human-comparison claim**; the unified/within-lexical result is stated at the
  internal-contrast-only floor for that arm. **No anchor is invented here.**
- **Q3 — reading rule:** instrument + C1–C4 inherited unchanged and re-frozen; the matched-kind
  certification and the survival-vs-collapse threshold sha256-frozen before the first probe call;
  survival and collapse reported symmetrically.

**Why this default makes a spurious result harder, not easier.** Option B holds the level fixed
and varies only the ambiguity kind, so it cannot manufacture a *survival* (which would revive the
defeated unified conjecture) by smuggling in a softer lexical stimulus — that is the very variable
it isolates. The matched-kind freeze blocks re-certifying which homonym contexts are "balanced"
against model hedging; the inherited C1 keeps the moment verdict decline-driven, not
confidence-driven; the Q2-b default refuses to lift the disjunctive lexical arm above
`internal-contrast-only` without a real, separately-ratified anchor — so the design cannot quietly
upgrade its own claim strength. **This choice is NOT made here; it is laid out for a future
independent adversarial-review pass.**

## Anchor / decision gap (explicit, surfaced not settled)

- **Open sub-question (Q2-a feasibility):** *Is there an in-repo or reachable sense-annotated
  homonym resource that certifies two distinct, co-licensed senses for a balanced-homonym
  context?* This page **does not assert one exists** and **invents no anchor**. If a future
  session finds and ratifies one (its own cross-session anchor decision), the disjunctive lexical
  class could rise from `internal-contrast-only` to a genuine sense-co-presence human-comparison
  claim — the one claim the DWUG route was explicitly forbidden. Until then the default is Q2-b.
- **Inherited-cap mismatch:** the resolved lexical anchor gate's usage-similarity cap is tied to
  the *scalar* DWUG stimulus and **does not transfer** to a disjunctive homonym class; any artifact
  must label the two lexical classes' anchor postures distinctly (usage-similarity-capped for the
  DWUG class; internal-contrast-only — or, if Q2-a ratifies, sense-co-presence — for the homonym
  class).

## Anti-cheat note

The **session boundary is load-bearing**: this page is **opened 2026-06-22 (session 85)**, no
follow-up result exists, and there is no result to motivate a choice. Ratification — at the
earliest by a *later* session via independent adversarial review with written rationale
(`resolved-by: autonomous (adversarial review)`) — fixes the **yardstick, never the result**
([`CLAUDE.md`](../../../CLAUDE.md) rule 6). The probe must **not** be run, nor the matched-kind
certification / disjunctive-class selection / survival-vs-collapse threshold re-tuned, in any
session that ratifies; a **pre-run critic NO-GO** against the *frozen* design **defers the run**
rather than relaxing any band. The named failure mode — **re-selecting the disjunctive lexical
class against model confidence** (the cross-kind analogue of the lexical anchor gate's "relabelling
post hoc the items the model happens to be unsure about") — is **forbidden by construction**: the
lexical disjunctive class and its kind-certification are frozen and sha256'd before any model
output. **Survival** (which would partly revive the defeated unified conjecture) must clear an even
higher bar than collapse, and both are reported symmetrically. The Q2 human-anchor obligation for
the disjunctive lexical class is **not** dodged here — it is a separate, deferred decision, and no
anchor is invented to lift the new class above `internal-contrast-only`.
