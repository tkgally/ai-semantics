# Essay: nothing to cancel

> **Status: draft (2026-06-28). A philosophical-track essay arguing in the project's own voice.** It introduces **no new empirical claim**: every empirical assertion cites the in-repo page that carries it, at that page's stated strength, and the original part is the argument — a conceptual distinction between two failure modes a *cancel* test can have. The essay **sharpens, does not refute**, the conjecture it leans on ([`conjecture/constructional-monotonicity-asymmetry`](../conjectures/constructional-monotonicity-asymmetry.md), `status: proposed`). It makes **no human comparison**: the two results it cites carry `anchor: internal-contrast-only`, and the B2 gate it rests on is a within-model design gate, never a human-comparison finding (the [`result/monotonicity-generalization-b2-nogo-v1`](../results/monotonicity-generalization-b2-nogo-v1.md) asymmetry was **never read**). One claim here leaned on a decision that was **resolved this session** by independent adversarial review ([`decisions/resolved/monotonicity-cancel-arm-redesign`](../../decisions/resolved/monotonicity-cancel-arm-redesign.md)); the resolution **affirms** the design-agnostic precondition this essay names (a redesigned cancel default must clear the B2 entailment-at-ceiling gate before any asymmetry is read), so that claim is no longer contingent — and the essay still does **not** name a specific redesign winner. The **Revision triggers** section states in advance what would weaken or strengthen the thesis; read it before citing the argument.

## The position

The conjecture's "cancellation is hard" is meant to name a competence gap: a construction is supposed to retract a verb's lexical default, and the model retracts it *less reliably* than it accumulates an added one. That is a claim about **defeasance** — about suppressing a default the model holds. This essay's bounded thesis is that **two very different things can make a "cancel" test hard, and conflating them corrupts the monotonicity reading**:

1. **Case (i) — a held entailment that fails to suppress.** The model *holds* the default as a (near-)entailment at ceiling on the bare frame, and *fails to suppress it* when the construction should cancel it. This is genuine defeasance difficulty. It is exactly what the conjecture's "cancellation is hard" is *supposed* to be about, and it is testable only when the default is a real entailment the cancel arm has to push against.
2. **Case (ii) — nothing to cancel.** The model *never held* the default as an entailment in the first place: it was only ever an implicature. The construction is then handed nothing to retract, the cancel arm is **unbuildable**, and any apparent "failure to cancel" is an artifact of an ill-posed test — **not** evidence about defeasance ability.

The distinction is sharp because the two look identical at the surface — both register as "the cancel arm did not behave" — but they license opposite readings. Case (i) is a finding about the model. Case (ii) is a finding about the *test*. A fair cancel test therefore requires a **held entailment as its target**; an *implicature* gives the construction nothing to cancel, so a cancel arm built on one cannot speak to defeasance at all.

This is a conceptual caution about a precondition, not a verdict on the conjecture. It does **not** say "cancellation is easy" or that the asymmetry is false. It says that before any cancel-arm number can count as evidence about defeasance, the arm must clear the case-(i) precondition: its default must be an entailment the model actually holds.

## The worked instance, at its actual strength

The session-135 B2 NO-GO is case (ii), and it was *correctly caught by the gate before any asymmetry was read*. The ratified cancel arm targeted the "only once" default of a semelfactive under the *for*-durative coercion. The gate's one-line finding, quoted exactly:

> "The gate is a NO-GO: the resultative add arm is buildable at ceiling (construction affirm 1.00, control 0.00 in 3/3 models), but the ratified *for*-durative cancel arm is not — its 'only once' default is affirmed at 0.00 in 3/3 models under both instruments. The failure is structural, not verb-selection: single-occurrence is a defeasible quantity implicature, not a lexical entailment, so a strict-NLI default can never reach the matched ceiling for any semelfactive." ([`result/monotonicity-generalization-b2-nogo-v1`](../results/monotonicity-generalization-b2-nogo-v1.md), §"One-line finding")

The diagnosis is the result's own, not the essay's: the default floored because it is an *implicature*, not an entailment. The result page draws the case-(ii) reading explicitly and refuses the case-(i) reading — this is the load-bearing distinction, in the result's own words:

> "the panel's 'neutral' verdict on 'only once' is **not** evidence about the models' *defeasance ability*; it is evidence that single-occurrence is **not an entailment** for these models, so there is no ceiling default to fail to suppress here." ([`result/monotonicity-generalization-b2-nogo-v1`](../results/monotonicity-generalization-b2-nogo-v1.md), §"What this does and does not license")

That is exactly the essay's case (i) / case (ii) cut, drawn by the result before the essay generalizes it. The asymmetry was **never read**: "The asymmetry was **never read** — the cancel-suppression cell was held out of the gate and the main battery was not run" (same section). And the result is `anchor: internal-contrast-only`: it "**Does not** make any human comparison (`internal-contrast-only`)" (same section). So the NO-GO is a within-model design gate firing, not a finding about how these models compare to people.

The contrast that makes the cut concrete is the cancel arm that *did* work. The conative cancels the transitive's completed-contact entailment, and contact **is** a near-ceiling lexical entailment — so the conative is a case-(i)-testable cancel arm. The de-confounding result states the contact default's ceiling:

> "The transitive lexical default is at ceiling (affirm-contact 91.7–100%), but **suppression with no cue** (the conative; `100 − conative_affirm`) is off-ceiling and model/instrument-dependent" ([`result/conative-cancel-direction-v2`](../results/conative-cancel-direction-v2.md), §"One-line finding")

The B2 NO-GO result names this same contrast as the reason its own failure is structural: "cc-v2's cancel direction worked because the transitive's contact entailment *is* a near-ceiling lexical entailment ('kicked the ball' ⊨ contact, 91.7–100%) that the conative then cancels" ([`result/monotonicity-generalization-b2-nogo-v1`](../results/monotonicity-generalization-b2-nogo-v1.md), §"Why this is structural, not a bad verb pool"). So the two arms differ in exactly the way the thesis predicts: the conative had a **held entailment** to push against (contact at 91.7–100%, case (i) — there was something to cancel, even if suppression was imperfect), while "only once" had only an **implicature** (floored at 0.00, case (ii) — there was nothing to cancel). The conative's *imperfect* suppression is the kind of datum the conjecture's "cancellation is hard" can be read off; "only once"'s floor is not.

Both cited results are `internal-contrast-only`; neither licenses a human comparison, and the essay makes none.

## The bounded positive reading (kept thin on purpose)

The s135 NO-GO is not *only* a measurement obstacle, and saying so keeps the essay honest. The panel's uniform "neutral" verdict on "only once" is a small, bounded **positive** datum: under strict NLI, these models **respect the implicature/entailment distinction** — they decline to over-commit a defeasible quantity implicature to "must be true." The result records the behavior precisely:

> "every cancel `default` item (flash, jump, cough, nod), in all three models, returned NLI label **1 (neutral)** — never **0 (entailment)** … The panel judges 'The light flashed' as **not entailing** 'The light flashed only once': a single flash is *consistent with* the sentence but not *forced* by it. Forced-choice agrees (0.00)." ([`result/monotonicity-generalization-b2-nogo-v1`](../results/monotonicity-generalization-b2-nogo-v1.md), §"Numbers (verified)")

This is a (thin, within-model, no-human-comparison) competence, not a deficit — but it must be framed at its true strength, which is small. It is **not** a claim that the models "understand" implicature, or that they reason about Gricean quantity. It is the **bounded behavioral observation** that they *label the implicature neutral*, exactly as a strict-NLI annotator should: the hypothesis is consistent with the premise but not forced by it, so "neutral" is the correct strict-NLI label. The competence claimed is no more than "these three models, under this instrument, on these four semelfactive verbs, do not mislabel a defeasible quantity implicature as an entailment." It is a within-model behavioral fact, single-run, small-N, and it is read off the very same floor that makes the cancel arm unbuildable. The same observation is, at once, a measurement obstacle for the cancel arm and a thin positive about the panel's calibration — and neither reading outruns the other.

## How this sharpens the conjecture (not a refutation)

"Cancellation is hard" must be **split into case (i) and case (ii)**. Only case (i) is the conjecture's claim — only a held entailment that fails to suppress is evidence about defeasance. Case (ii) is not a weak version of the conjecture; it is *outside its scope*, an ill-posed test that the conjecture's own discipline is built to exclude. The conjecture already demands "matched difficulty / ceiling control" in every confirm test (its §"What would confirm / falsify"), and the ratified operationalization turned that demand into the B2 default-at-ceiling gate — which is **exactly** the case-(i) precondition made mechanical. The gate's whole job is to certify, before any spend, that the cancel default is a real entailment at ceiling on the bare frame; an arm whose default is an implicature fails the gate *because* it is not case-(i)-testable. The conjecture page itself now records this lesson:

> "part of what reads as 'cancellation is hard' may be that the cancelled inferences the project reaches for are implicatures the models never held as entailments (weigh-question 4 of that decision)." ([`conjecture/constructional-monotonicity-asymmetry`](../conjectures/constructional-monotonicity-asymmetry.md), §"Decisive test BUILT; cancel arm hit a B2 NO-GO …")

The essay's contribution is to make that the *standing* split rather than a one-off note: every cancel arm the project builds must first be shown to target a held entailment, or its "failure to cancel" cannot be read as defeasance difficulty. And there is a second, finding-shaped consequence. The project's recurring trouble building cancel arms is **also a map of which defaults these models treat as entailments versus implicatures**. Contact (from the transitive): held as an entailment (91.7–100%). Single-occurrence (from a semelfactive): not held — an implicature (floored at 0.00). That map is a small within-model finding about the models' semantic competence in its own right — the cancel-arm difficulty is not only an engineering nuisance but a probe of where the entailment/implicature line falls for these models — and it is exactly what a careful case-(i)/case-(ii) audit produces as a by-product.

**A precondition the redesign decision now makes binding, not a redesign verdict.** The general lesson is that *any* redesigned cancel default must clear the B2 entailment-at-ceiling gate to be case-(i)-testable. That is the essay's point and it is design-agnostic. The essay does **not** prejudge how the cancel arm should be redesigned: the choice between the options on the table — telic-completion, privative modifiers, a stronger same-frame indicator, or a principled-limit closure — was **resolved this session** ([`decisions/resolved/monotonicity-cancel-arm-redesign`](../../decisions/resolved/monotonicity-cancel-arm-redesign.md): ADOPT C1 telic-completion conditional on its own B2 calibration, C2 privative fallback, then a closure if both fail) and was *that decision's job, not this essay's*. The decision-agnostic statement the essay makes — whatever cancel coercion is chosen, it must be **measured** against the B2 default-at-ceiling gate before any asymmetry is read — is exactly what the resolution makes binding ("do **not** repeat the s135 trap by assuming a new default is at ceiling; **measure it**", §"The named failure mode this gate guards"). The essay names no winner; the resolution's conditional adoption of C1 turns on precisely this gate.

## Sibling note

This essay shares the add/cancel parent observation with several siblings and cuts a different axis from each. It is about a trap on the **cancel** side — the entailment-vs-implicature status of the cancel *target* — which none of the siblings names.

- [`essay/add-cancel-separability`](add-cancel-separability.md) is the closest sibling and the right one to differentiate against. It argues that the two **poles** (add and cancel) are separable only as "an operational achievement of careful item construction, not a pre-given joint in distributional meaning" — its worry lives chiefly on the **add** side (the control must not already license the would-be-added entailment; the default-coincidence trap, generalized from AANN). This essay's worry lives on the **cancel** side and is a different question: not "are the two poles separable?" but "does the cancel arm even *have a target* — a held entailment — to cancel?" The two are complementary preconditions, one per pole: separability earns the add arm's headroom; a held entailment earns the cancel arm's defeasance test. Where `add-cancel-separability` says the add arm needs a control that is *off-ceiling*, this essay says the cancel arm needs a default that is *on-ceiling* (and an entailment, not merely a high-probability implicature) — the mirror-image precondition. (Note the asymmetry of the two preconditions is itself instructive: the s135 add arm passed its ceiling check and the cancel arm failed its, for opposite reasons — the add control must *lack* the entailment, the cancel default must *hold* it.)
- [`essay/predictive-objective-favors-accumulation`](predictive-objective-favors-accumulation.md) reads the *direction* of the asymmetry as the fingerprint of a next-token predictive objective — *assuming the asymmetry is real and general*. That essay is downstream of a well-posed cancel arm: it takes the add/cancel difference as a measured thing and reads a direction off it. This essay is upstream: it asks whether a given cancel arm is well-posed *at all* (case (i)) or ill-posed (case (ii)) before any direction can be read. A case-(ii) cancel arm gives that essay no signal to interpret. Cited to mark the boundary; this essay makes no claim about the asymmetry's direction.
- [`essay/inference-default-coincidence`](inference-default-coincidence.md) is the converse measurement trap on the **add** side: there the construction's licensed inference *coincides with* the phrase's distributional default, so the add arm has no headroom ("the default eats the construction"). That is an add-side coincidence (default = the would-be-added inference). This essay's case (ii) is a cancel-side **status** problem (the would-be-cancelled default is only an implicature, so there is nothing to retract). Both are "the test cannot reach its question," but at opposite poles and for different reasons; cited to mark the boundary, not re-derived.

## Revision triggers (read before citing)

The thesis is conditional and arm-relative; the following concrete future evidence would weaken or strengthen it.

- **(a) A redesigned cancel arm whose default DOES clear B2 and then DOES show a defeasance gap.** If a future cancel arm targets a genuine held entailment (clears the B2 default-at-ceiling gate, ≥ 0.80 in ≥ 2/3 models on the bare frame) and *then* the construction suppresses it less reliably than the add arm accumulates — that is **case (i) confirmed**, and the thesis is **vindicated as a caution that was met**: the cancel arm was made well-posed, and defeasance difficulty was read off it. The essay moves to `status: revised`, recording that the case-(i) precondition was discharged for that battery. (This does not by itself settle the conjecture, only that its cancel arm was finally testable.)
- **(b) A redesigned cancel default that ALSO floors at B2.** If the next cancel default (e.g. telic-completion, or any other coercion the redesign reaches for) *also* fails the entailment-at-ceiling gate — read by the models as neutral, not entailment — that **strengthens** the broader pattern this essay names: the project keeps reaching for *implicatures* (single-occurrence, perhaps completion) when it wants entailments to cancel, so the case-(ii) trap is recurrent, not a one-off. The essay would then sharpen to "the project's natural cancel targets are disproportionately implicatures," and the map-of-entailments-vs-implicatures by-product grows another data point.
- **(c) A within-frame re-pair that somehow lifts "only once" to ceiling.** The s135 result calls the floor structural ("no choice of semelfactive verb can lift this default to the ≥ 0.80 ceiling", §"Why this is structural, not a bad verb pool"). If some semelfactive verb pool *did* reach ceiling on "only once" under strict NLI, the worked instance would be reclassified (the floor would be verb-selection after all, not structural), and the essay's reliance on s135 as a clean case-(ii) example would need rechecking against that evidence — though the general case-(i)/case-(ii) distinction would survive.
- **(d) An in-repo source on the entailment/implicature or defeasance boundary.** The entailment-vs-implicature cut is used here at the level the result pages state it (strict-NLI "must be true" vs "consistent with but not forced by"). If a semantics/pragmatics source enters the repo and gives that cut a more principled basis — or complicates it (e.g. semelfactive single-occurrence as a stronger default than "mere implicature") — the essay re-checks that "implicature, so nothing to cancel" is stated against the correct target and relativizes accordingly.
- **(e) Representational evidence on whether a default is "held."** This essay is behavioral: "held as an entailment" means "labeled entailment at ceiling under strict NLI." If a mechanistic probe were to show a model internally *represents* a single-occurrence default even while labeling it neutral, then "the model never held it" would be relativized to "the model does not behaviorally commit to it," and case (ii) would tighten to "behaviorally unbuildable," leaving open a representational version of the cancel target.

## What this essay is not

- **Not a claim that cancellation is easy, or that the asymmetry is false.** It is a caution about a precondition — the cancel target must be a held entailment — not a verdict on the conjecture, which it **sharpens** by splitting "cancellation is hard" into case (i) (in scope) and case (ii) (out of scope, ill-posed).
- **Not a claim that the models "understand" implicature.** The bounded positive is only that the panel *labels* a defeasible quantity implicature neutral under strict NLI, exactly as a strict-NLI annotator should — a within-model behavioral fact, not an attribution of pragmatic reasoning.
- **Not a redesign verdict.** The choice of replacement cancel coercion was **resolved this session** ([`decisions/resolved/monotonicity-cancel-arm-redesign`](../../decisions/resolved/monotonicity-cancel-arm-redesign.md): ADOPT C1 telic-completion conditional on its own B2 calibration, C2 fallback); the essay names only the design-agnostic precondition (any redesigned default must clear the B2 entailment-at-ceiling gate) — which the resolution makes binding — and does not itself pick a winner.
- **Not a human-comparison claim.** Both cited results are `anchor: internal-contrast-only`; the B2 gate is a within-model design gate, and the s135 asymmetry was never read. No claim here is set against any human norm.
- **Not a tested claim about the entailment/implicature boundary itself.** That cut is used at the strength the result pages state it (strict-NLI "must be true"); the essay does not upgrade it to a formal semantic result.

## Honesty box
